    What is a web crawler.
    Feb. 2, 2021 · 4 min · Greg Conmajia

    Web crawlers, usually known as web spiders, are automated data samplers and remote content fetchers. These programmed agents retrieve web contents and store them for future analytic usage. A typical crawler loads web pages from the internet, abstracts contents of specified fields in page HTML sources, and constructs these data into a JSON object or other formats of storage. In this scenario, the crawler substitutes web browsers operated by human clients.

    As shown in figure 1, a web crawler disguises the role of a human-controlled web browser. Instead of surfing via the browser, the web crawler obtains a full access to the internet, and, consequentially, presses all data retrieved from the cloud into storages.

    Example Web Crawler#

    A simplest web crawler can be implemented with a single feature, which is to retrieve web data from the internet. No future data processors or analyzers attached except a simple regular expression extractor for the page title. One of the most commonly used libraries for this job is requests.

    Note that a request header, which can be found in development tools of all modern web browsers, must be employed to disguise this web crawler as a regular web browser client, or the anti-crawling server may return a 418 error instead of demanding web pages.

    Let’s take Amazon’s website for example, sample response will probably resemble the following fragment (crawled on 3rd Feb. 2021):

    import requests
url = "https://www.amazon.com"
header = {
    "user-agent": "Mozilla/5.0 (Macintosh; Intel Mac OS X ... Safari/605.1.15"
}
# retrieve content at given location with the request header
response = requests.get(url, headers = header)
print(response.text)

    So far, we’ve succeeded downloading, or more popularly, crawling contents from the internet. Further fancy procedures are now able to be applied on the data stored in response.text.

    Data Extraction#

    It is obvious that crawling web pages from the internet makes the web crawler no more than a dumb browser. Data hidden underneath the bundle of HTML source text shall be extracted and well-organized for the next phase of a real practical web crawler.

    Plenty of data extractors are available out there, depending on the format of content, e.g. Beautiful Soup, XPath, JSONPath, etc. Almost every extractor library utilizes the solo powerful yet complicated tool - regular expressions, or RegEx - to a certain extent.

    Python provides the ability of manipulating regular expressions through a library named re. In this section, I am implementing a simple extractor with the re library, directly draw desired content from the whole. Fundamental concepts and details on regular expressions can be googled and will not be discussed in this article.

    While implementing extractors with re, the desired data and its wrapping structures shall be studied ahead. In this case, I’m going to find the title of what’s retrieved, regardless of the source of web pages. Since every page claims its title in a <title> tag in the head section, it is quite convenient to build the target regular expression with group matches.

    <title>(.*?)</title>

    A non-greedy matching group (.*?) is employed to extract the desired text itself without its wrapper. Therefore, the Python code of such extractor will be:

    import re
pattern = "<title>(.*?)</title>"
match = re.search(pattern, response.text)
print(match.group(1))

    Amazon.com: Online Shopping for Electronics, Apparel, Computers, Books, DVDs & more

    And the title of the page crawled, from the previous Amazon’s homepage, is unveiled.

    Definitely, text-based regular expressions are arduous to master. Some DOM-based1 solutions, as mentioned beforehand, could be a better and more effective choice for new programmers.

    Data Storage and Analysis#

    After everything is ready extracted, under most circumstances, the data is stored in either structured data objects/variables, or in a designated database. Specified analysis will then be carried out over these data, to study deepened characteristics or patterns of the original data source. The entire “fetch-extract-store-analyze” procedure makes the Python codes a “web crawler”.
