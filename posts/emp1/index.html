    • Natural-Imitating Random Number Generator

    Generates random natural variables in a relative continuity.

    Random number generator, or RNG, is one of the most fundamental ingredients of all computers and programming languages, in the form of electronic or software modules. RNGs were invented long before mechanical or electronic computers before there were symbols for writing numbers.

    Ancient civilizations used flipping coins or rolling dices to generate independent random numbers. The most antique dices found in Iraq and Iran were made 5,000 years ago. They were also popular in India and China since 2,000 BC (Diro, 2017). Modern RNGs are equipped often with a uniformly distributed white noise generator as the source of randomness, which, as a characteristic, provides extremely discrete outputs.

    Under most circumstances, this is the exact requirement for RNGs. But, yes there is a but, when simulating continuous natural measurements, such as wind speed, water level, temperature, etc., the uniformity becomes less important, or even an impediment. In this case, we will need a smoother RNG, which introduces my project, a natural-imitating random number generator, the NiRNG. By the phrase “natural imitating”, continuous measurements are concerned.

    Continuity of Measurements in Nature#

    Values of measurements relating to nature, temperature readouts on a thermometer, for example, do not change discretely. Say, it is 80℉ now, we don’t expect it be 100℉ in the next second, even if heaters were turned on. These values always change gradually, continuously. However, all random number generator outputs vary extremely quickly, as getting rid of the bias of a random bit sequence is one of the primary tasks of designing RNGs. No next values can be inferred from previous ones. Low correlation guarantees randomness of RNGs, sacrificing continuity of values.

    Of course there is quite a chance to guess outputs. $v=v_0\pm\tau p$, where $v_0$ is the latest output, $\tau$ is the increment range size, and $p$ is a RNG produced randomized factor.

    Uniformly Distributed RNs vs. Natural-Imitating RNs#

    When we talk about natural-imitating stuffs, we intend to reproduce real world behaviors in our desired environments, or at least, to simulate natural phenomenons. With such a great goal, some foundations must be set, one of which is the natural-imitating random number generator.

    Figure 1 presents a visible comparison of two random number sequences generated by a uniformly distributed RNG and a NiRNG. The chart explains the main purpose of the NiRNG: generate random numbers in a relatively continuous manner. The NiRNG data seems much smoother than that of the RNG with a bit of jitters and a recognizable trend in certain neighborhoods while its rival provides wild sudden changes from the very bottom up to the ceiling. Unfortunately, the messed RNG solution is the choice of quite a lot of amateurs and noob engineers, giving gauge hands and meter indicators their most horrified nightmares.

    As shown, differences $\delta_k$ between two adjacent values in the NRNG sequence is highly restricted in a small amount. For this particular test data set, the correlation coefficient of RNG $r_\text{RNG}=0.005$, while $r_\text{NRNG}=0.91$, indicating a very high degree of correlation of adjacent data.

    Despite of the high bias of the random bits, resulting output data of the NiRNG solution provides a local relatively high randomness in the neighborhood of a bandwidth defined by range of incrementation.

    Design of NiRNG#

    Figure 2 shows the flow diagram. Four ranges are employed in this design:

    1. regular range
    2. increment range
    3. primary nominal range
    4. secondary nominal range

    The later two are elected internally by the initial value of output as a final nominal range. A short sequence of minor jitters is appended to the output as spikes.

    Take the example in the last section, fig.3 shows its nominal range (elected from primary and secondary nominal ranges) between blue horizontal lines.

    Any output value is based on its prior predecessor. A random increment is generated each time NiRNG outputs and is accumulated onto the previous value. In fact, the NRNG generates random numbers in a much narrower band where randomness is limited, but former output information is kept to guarantee that final output (at any step) be relatively continuous, which makes the most distinction from what ordinary RNGs generate.

    Software Implementation#

    Embracing the power of R: This implementation of NiRNG is writtin in R language, one of the most popular statistical programming languages.

    Core NiRNG algorithm consists of three main procedures: a) determine actual nominal range, b) generate randomized spikes, and c) accumulate random increment onto current value. As implemented in R as follows.

    # A simplified approach of NiRNG
# 8 Jan. '21 Greg Conmajia
if (weight.nominal.switch > 0)
    if (value >= min.nominal && value <= max.nominal)
        if (rand() <= weight.nominal.switch)
            if (min.nominal == min.nominal.primary
             && max.nominal == max.nominal.primary) {
                min.nominal <- min.nominal.secondary
                max.nominal <- max.nominal.secondary
            } else {
                min.nominal <- min.nominal.primary
                max.nominal <- max.nominal.primary
            }

if (weight.spike > 0)
    if (rand() <= weight.spike)
        length.spike.remain <- length.spike

if (length.spike.remain == 0) {
    if (rand() > weight.direction)
        direction <- direction * -1
    if (value < min.nominal)
        if (rand() < weight.nominal)
            direction <- 1
    if (value >= max.nominal)
        if (rand() < weight.nominal)
            direction <- -1
} else
    length.spike.remain <- length.spike.remain - 1

if (value <= minimum)
    direction <- 1
if (value >= maximum)
    direction <- -1
value <- value + direction * rand(min.increment * 100, max.increment * 100) / 100
value <- max(value, minimum)
value <- min(value, maximum)

    Above code generates a single numeric output each time, based on given parameters and the noise source rand() could be any type of random number sources, which, in this case, is a function of runif.

    # Generates a single random number
rand <- function (min = 0, max = 100) {
  return(runif(1, min = min, max = max))
}

    With the use of true noise source hardware, we are enabled to implement NiRNG in the physical world. Noise diodes, heated electron tubes are appropriate cost effective candidates of such sources for homemade projects.
